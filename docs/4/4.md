# Режим измерения (Tracking)

Основной рабочий режим датчика INA — накопление данных о потреблённой энергии.

## Запуск

- **Веб-интерфейс:** кнопка **Track** на вкладке датчика → подтверждение
- **API:**
  ```bash
  curl "192.168.241.177/start?serial=WH0147641"
  ```

## Измеряемые величины

| Величина | Единица | Описание |
|----------|---------|----------|
| `currentCurrent` | mA | Текущий ток |
| `currentVoltage` | V | Текущее напряжение |
| `currentPower` | mW | Текущая мощность |
| `totalCurrent` | mAh | Накопленный заряд |
| `totalPower` | mWh | Накопленная энергия |
| `maxCurrent` | mA | Пиковый ток |
| `maxPower` | mW | Пиковая мощность |
| `minVoltage` | V | Минимальное напряжение (просадка) |
| `totalTime` | сек | Время измерения |

Накопление (дискретное интегрирование с интервалом опроса $\Delta t$):
```
elapsedHours = elapsedTime / 3600000
totalCurrent += currentCurrent × elapsedHours
totalPower   += currentPower   × elapsedHours
```

Интервал опроса по умолчанию — 500 мс (настраивается в Circuit params).

## Остановка

- **Веб-интерфейс:** кнопка **Stop**
- **API:**
  ```bash
  curl "192.168.241.177/stop?serial=WH0147641"
  ```
  Результат возвращается в ответе, в файл **не** сохраняется
- **Автоматически:** при переходе в настройки или потере соединения с датчиком (данные теряются)

## Дисплей

<img src="../4/tracking.png" width="500">

Отображается: время измерения, напряжение, ток, накопленный заряд (красный), накопленная энергия (оранжевый), пиковый ток/мощность, мин. напряжение.

## LED

При активном измерении на любом датчике — быстрое мигание (300/300 мс).

# Measurement Mode (Tracking)

The primary operating mode of the INA sensor — accumulating consumed energy data.

## Starting

- **Web interface:** **Track** button on the sensor tab → confirmation
- **API:**
  ```bash
  curl "192.168.241.177/start?serial=WH0147641"
  ```

## Measured Quantities

| Quantity | Unit | Description |
|----------|------|-------------|
| `currentCurrent` | mA | Instantaneous current |
| `currentVoltage` | V | Instantaneous voltage |
| `currentPower` | mW | Instantaneous power |
| `totalCurrent` | mAh | Accumulated charge |
| `totalPower` | mWh | Accumulated energy |
| `maxCurrent` | mA | Peak current |
| `maxPower` | mW | Peak power |
| `minVoltage` | V | Minimum voltage (sag) |
| `totalTime` | s | Measurement time |

Accumulation (discrete integration with polling interval $\Delta t$):
```
elapsedHours = elapsedTime / 3600000
totalCurrent += currentCurrent × elapsedHours
totalPower   += currentPower   × elapsedHours
```

Default polling interval — 500 ms (configurable in Circuit params).

## Stopping

- **Web interface:** **Stop** button
- **API:**
  ```bash
  curl "192.168.241.177/stop?serial=WH0147641"
  ```
  The result is returned in the response and is **not** saved to a file
- **Automatically:** when entering settings or losing connection to the sensor (data is lost)

## Display

<img src="../4/tracking.png" width="500">

Displays: measurement time, voltage, current, accumulated charge (red), accumulated energy (orange), peak current/power, min voltage.

## LED

Fast blinking (300/300 ms) when any sensor has an active measurement.
