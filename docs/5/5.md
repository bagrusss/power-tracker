# Конечные автоматы (State Machines)

## Состояния МК

`McStateMachine` — 6 состояний (`StateType`).

```
[Старт]
   │
   ▼
ConnectingWifi ── успех ──► Idle ── Settings ──► Idle
   │                ▲         │                    │
   └── нет SSID ──►SetupWifi─┘                    │
                                                   │
                    потеря WiFi ──► ConnectingWifi ◄┘

OtaUpdating ◄── из любого состояния
```

| Состояние | Класс | Описание |
|-----------|-------|----------|
| `ConnectingWifi` | `ConnectingWifiState` | Подключение к WiFi. Неудача → AP mode → Idle. Нет SSID → SetupWifi |
| `Idle` | `IdleState` | Ожидание. Вкладки датчиков, LED. Кнопка Settings |
| `SetupWifi` | `SetupWifiState` | Первичный ввод SSID/пароля. Skip → Idle, Apply → reboot |
| `Settings` | `SettingsState` | WiFi, Board, OTA, параметры датчиков. Apply → Idle/reboot |
| `Warning` | `WarningState` | Заглушка |
| `OtaUpdating` | `OtaUpdatingState` | Прогресс OTA на дисплее |

### Переходы

| Из | В | Условие |
|----|----|---------|
| ConnectingWifi | Idle | WiFi подключён или создана AP |
| ConnectingWifi | SetupWifi | Нет сохранённых SSID/пароля |
| Idle | Settings | Кнопка Settings |
| Idle | ConnectingWifi | Потеря WiFi |
| Settings | Idle | Apply без изменений |
| Settings | reboot | Изменены WiFi или параметры цепи |
| SetupWifi | Idle | Skip |
| SetupWifi | reboot | Apply |
| Любое | OtaUpdating | Запуск OTA |

---

## Состояния датчиков INA

`InaStateMachine` — 2 состояния на каждый датчик (`InaStateType`).

```
Idle ── Track/API start ──► Tracking
  ▲                            │
  └──── Stop/API stop ─────────┘
```

| Состояние | Класс | Описание |
|-----------|-------|----------|
| `Idle` | `InaIdleState` | Мгновенные значения: V, mA, mW, mV на шунте |
| `Tracking` | `InaTrackingState` | Накопление mAh/mWh, пики, мин. напряжение |

### Переходы

| Из | В | Условие |
|----|----|---------|
| Idle | Tracking | Кнопка Track или API `start` |
| Tracking | Idle | Кнопка Stop (сохраняет в файл) или API `stop` (без сохранения) |
| Tracking | Idle | Выход из состояния (смена вкладки, настройки) — сохраняет в файл |

### Уведомления

`InaStateMachine::switchState` → `InaStateMachines` → `notifySensorStatusChanged` → LED-индикация, дисплей.

---

## Взаимодействие

```
McStateMachine
  └── IdleState.handle() → InaStateMachine.handle()

InaStateMachines
  ├── init() — создаёт InaStateMachine для каждого датчика
  ├── getInaStateMachine(addr) / getInaStateMachine(serial)
  ├── isAnySensorBusy() — true если есть Tracking
  └── getAllSensorsInfo() — JSON

CommandsHandler (API)
  ├── start(id) → InaStateMachine.switchState(Tracking)
  ├── stop(id)  → InaStateMachine.switchState(Idle)
  ├── status(id) → getCurrentState().printStatus()
  └── all()     → getAllSensorsInfo()
```

# State Machines

## MCU States

`McStateMachine` — 6 states (`StateType`).

```
[Start]
   │
   ▼
ConnectingWifi ── success ──► Idle ── Settings ──► Idle
   │                ▲           │                    │
   └── no SSID ───►SetupWifi───┘                    │
                                                    │
                     WiFi lost ──► ConnectingWifi ◄──┘

OtaUpdating ◄── from any state
```

| State | Class | Description |
|-----------|-------|----------|
| `ConnectingWifi` | `ConnectingWifiState` | WiFi connection. Failure → AP mode → Idle. No SSID → SetupWifi |
| `Idle` | `IdleState` | Standby. Sensor tabs, LED. Settings button |
| `SetupWifi` | `SetupWifiState` | Initial SSID/password entry. Skip → Idle, Apply → reboot |
| `Settings` | `SettingsState` | WiFi, Board, OTA, sensor parameters. Apply → Idle/reboot |
| `Warning` | `WarningState` | Stub |
| `OtaUpdating` | `OtaUpdatingState` | OTA progress on display |

### Transitions

| From | To | Condition |
|----|----|---------|
| ConnectingWifi | Idle | WiFi connected or AP created |
| ConnectingWifi | SetupWifi | No saved SSID/password |
| Idle | Settings | Settings button |
| Idle | ConnectingWifi | WiFi lost |
| Settings | Idle | Apply without changes |
| Settings | reboot | WiFi or circuit parameters changed |
| SetupWifi | Idle | Skip |
| SetupWifi | reboot | Apply |
| Any | OtaUpdating | OTA started |

## INA Sensor States

`InaStateMachine` — 2 states per sensor (`InaStateType`).

```
Idle ── Track/API start ──► Tracking
  ▲                            │
  └──── Stop/API stop ─────────┘
```

| State | Class | Description |
|-----------|-------|----------|
| `Idle` | `InaIdleState` | Instantaneous values: V, mA, mW, mV on shunt |
| `Tracking` | `InaTrackingState` | Accumulation of mAh/mWh, peaks, min voltage |

### Transitions

| From | To | Condition |
|----|----|---------|
| Idle | Tracking | Track button or API `start` |
| Tracking | Idle | Stop button (saves to file) or API `stop` (without saving) |
| Tracking | Idle | Exiting the state (tab switch, settings) — saves to file |

### Notifications

`InaStateMachine::switchState` → `InaStateMachines` → `notifySensorStatusChanged` → LED indication, display.

---

## Interaction

```
McStateMachine
  └── IdleState.handle() → InaStateMachine.handle()

InaStateMachines
  ├── init() — creates InaStateMachine for each sensor
  ├── getInaStateMachine(addr) / getInaStateMachine(serial)
  ├── isAnySensorBusy() — true if Tracking is active
  └── getAllSensorsInfo() — JSON

CommandsHandler (API)
  ├── start(id) → InaStateMachine.switchState(Tracking)
  ├── stop(id)  → InaStateMachine.switchState(Idle)
  ├── status(id) → getCurrentState().printStatus()
  └── all()     → getAllSensorsInfo()
```
