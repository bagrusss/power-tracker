# Графы состояний (State Machines)

## Состояния микроконтроллера (МК)

МК реализован на [`McStateMachine`](../../src/states/mc/McStateMachine.cpp) и имеет 6 состояний, определённых в [`StateType`](../../src/states/mc/State.h:10).

### Диаграмма переходов

```
                    ┌──────────────────────────────────────┐
                    │              [Старт (setup)]          │
                    └──────────────────┬───────────────────┘
                                       │
                                       ▼
                    ┌──────────────────────────────────────┐
                    │          ConnectingWifi              │
                    │  (попытка подключиться к WiFi)       │
                    └──────┬───────────────┬───────────────┘
                           │               │
                    ┌──────┴──────┐  ┌─────┴──────────┐
                    │  Успех     │  │  Нет SSID      │
                    ▼             ▼  ▼                │
           ┌────────────────┐ ┌──────────────────┐    │
           │     Idle       │ │   SetupWifi      │◄───┘
           │ (режим ожидания)│ │ (настройка WiFi) │
           └───┬────┬───────┘ └──────────────────┘
               │    │
               │    │ кнопка "Settings"
               │    ▼
               │  ┌──────────┐
               │  │ Settings │── Apply ──► Idle
               │  │(настройки)│── reboot ──► [перезагрузка]
               │  └──────────┘
               │
               │ потеря WiFi (колбэк onWifiDisconnected)
               └──────────────────────────────────► ConnectingWifi

  ┌──────────────────┐
  │   WarningState   │ (зарезервировано, не реализовано)
  └──────────────────┘

  ┌──────────────────┐
  │  OtaUpdating     │ ◄── из любого состояния при старте OTA
  │  (обновление)    │
  └──────────────────┘
```

### Таблица состояний МК

| Состояние | Класс | Вход (onEnter) | Выход (onLeft) | Обработчик (handle) |
|-----------|-------|----------------|----------------|---------------------|
| `ConnectingWifi` | [`ConnectingWifiState`](../../src/states/mc/ConnectingWifiState.cpp) | Включает LED | Выключает LED | Пытается подключиться к WiFi. При успехе → Idle. При неудаче → Idle (AP mode). Если нет SSID → SetupWifi |
| `Idle` | [`IdleState`](../../src/states/mc/IdleState.cpp) | Включает LED (медленное мигание), обновляет активное устройство | Выключает LED | Обрабатывает активный датчик INA (его handle) |
| `SetupWifi` | [`SetupWifiState`](../../src/states/mc/SetupWifiState.cpp) | — | — | Отображает форму ввода SSID/пароля. Apply → reboot. Skip → Idle |
| `Settings` | [`SettingsState`](../../src/states/mc/SettingsState.cpp) | Включает LED (постоянно) | Выключает LED, если были изменения → reboot | Отображает меню настроек |
| `Warning` | [`WarningState`](../../src/states/mc/WarningState.cpp) | Пусто | — | Пусто (заглушка) |
| `OtaUpdating` | [`OtaUpdatingState`](../../src/states/mc/OtaUpdatingState.cpp) | Устанавливает колбэк OTA, выводит "Installing..." на дисплей | Сбрасывает колбэк OTA | Отображает прогресс установки на дисплее |

### Условия переходов

| Из | В | Условие |
|----|----|---------|
| ConnectingWifi | Idle | Успешное подключение к WiFi |
| ConnectingWifi | Idle | Неудачное подключение (поднята AP) |
| ConnectingWifi | SetupWifi | Нет сохранённых SSID и пароля |
| Idle | Settings | Нажата кнопка "Settings" |
| Idle | ConnectingWifi | Потеря WiFi-соединения (колбэк `onWifiDisconnected`) |
| Settings | Idle | Нажата кнопка "Apply" (без изменений WiFi/цепи) |
| Settings | reboot | Были изменения WiFi или параметров цепи |
| SetupWifi | Idle | Нажата кнопка "Skip" |
| SetupWifi | reboot | Нажата кнопка "Apply" |
| Любое | OtaUpdating | Запуск OTA-обновления (через OtaUpdater) |

## Состояния датчиков INA

Каждый датчик INA имеет собственный конечный автомат [`InaStateMachine`](../../src/states/ina/InaStateMachine.cpp) с 2 состояниями, определёнными в [`InaStateType`](../../src/states/ina/InaState.h:12).

### Диаграмма переходов

```
                    ┌──────────────────────────────────────┐
                    │           [Старт (init)]             │
                    └──────────────────┬───────────────────┘
                                       │
                                       ▼
                    ┌──────────────────────────────────────┐
                    │              Idle                     │
                    │  (отображение мгновенных значений)    │
                    └──────┬───────────────────┬───────────┘
                           │                   │
                    ┌──────┴──────┐    ┌───────┴──────────┐
                    │  Кнопка    │    │  API stop         │
                    │  "Track"   │    │  (из Tracking)    │
                    │  API start │    │                   │
                    ▼             │    ▼                   │
           ┌──────────────────────┴──────────────────┐     │
           │              Tracking                    │────┘
           │  (активное измерение с накоплением)      │
           └─────────────────────────────────────────┘
```

### Таблица состояний датчика INA

| Состояние | Класс | Вход (onEnter) | Выход (onLeft) | Обработчик (handle) |
|-----------|-------|----------------|----------------|---------------------|
| `Idle` | [`InaIdleState`](../../src/states/ina/InaIdleState.cpp) | — | — | Отображает мгновенные значения: V, mA, mW, mV на шунте |
| `Tracking` | [`InaTrackingState`](../../src/states/ina/InaTrackingState.cpp) | Сбрасывает счётчики, устанавливает `isTracking = true` | Останавливает измерение (stop), сохраняет результаты в файл | Опрашивает датчик с интервалом, накапливает totalCurrent/totalPower, отслеживает пики |

### Условия переходов

| Из | В | Условие | Действие при переходе |
|----|----|---------|----------------------|
| Idle | Tracking | Нажата кнопка "Track" в веб-интерфейсе | Сброс счётчиков, старт измерения |
| Idle | Tracking | API-запрос `start?serial=...` | Сброс счётчиков, старт измерения |
| Tracking | Idle | Нажата кнопка "Stop" в веб-интерфейсе | Остановка измерения, сохранение результатов в `/results_<addr>.json` |
| Tracking | Idle | API-запрос `stop?serial=...` | Остановка измерения, возврат результатов в ответе (без сохранения в файл) |
| Tracking | Idle | Выход из состояния (onLeft) при переключении вкладки/состояния МК | Остановка измерения, сохранение результатов |

### Уведомления

При каждом переходе состояния датчика INA [`InaStateMachine::switchState`](../../src/states/ina/InaStateMachine.cpp:23) уведомляет [`InaStateMachines`](../../src/states/ina/InaStateMachines.cpp:100), который оповещает всех подписчиков (`SensorStatusListener`) через `notifySensorStatusChanged`. Это используется для обновления LED-индикации и отображения статуса на дисплее.

## Взаимодействие конечных автоматов

```
McStateMachine (МК)
    │
    ├──► handle() ──► IdleState.handle() ──► InaStateMachine.handle() (активный датчик)
    │
    ├──► buildUI() ──► IdleState.buildUi() ──► InaStateMachine.buildUI() (активный датчик)
    │
    └──► updateUi() ──► IdleState.updateUi() ──► InaStateMachine.updateUi() (активный датчик)

InaStateMachines (менеджер всех датчиков)
    │
    ├──► init() ──► создаёт InaStateMachine для каждого обнаруженного датчика
    │
    ├──► getInaStateMachine(address) ──► возвращает автомат по адресу
    │
    ├──► getInaStateMachine(serial) ──► возвращает автомат по серийному номеру
    │
    ├──► isAnySensorBusy() ──► true, если хотя бы один датчик в Tracking
    │
    └──► getAllSensorsInfo() ──► JSON со статусами всех датчиков

CommandsHandler (API)
    │
    ├──► handleStart(id) ──► InaStateMachine.switchState(Tracking)
    ├──► handleStop(id)  ──► InaStateMachine.switchState(Idle)
    ├──► handleStatus(id)──► InaStateMachine.getCurrentState().printStatus()
    └──► handleAll()     ──► InaStateMachines.getAllSensorsInfo()
```
