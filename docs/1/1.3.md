# Настройка и калибровка датчиков

Количество вкладок соответствует числу обнаруженных датчиков при старте. Общие вкладки (Wi-Fi, Board, Updates) видны всегда, даже без датчиков.

## Wi-Fi settings

<img src="../1/wifi-change.png" width="400">

Смена SSID/пароля. ESP8266 — только 2.4 ГГц. При сохранении — перезагрузка.

## Board config

Частота шины I2C: от 100 кГц (по умолчанию) до 400 кГц. При изменении — перезагрузка.

## Updates

1. Указать URL к `ota.json`:
```json
{
  "version": "1.0.0",
  "buildNumber": 2,
  "esp8266": "http://example.com/ota/esp_8266_firmware.bin"
}
```
Ключ платформы: `esp8266` или `esp32` (строчные, без пробелов). URL бинарника — без редиректов.

2. Выбрать интервал проверки
3. Включить автообновление

МК сравнивает `buildNumber`. Если выше — скачивает и прошивает. При активном измерении на любом датчике OTA не запустится.

## Device info

Настройки наблюдаемого прибора для каждого датчика: название, ОС, серийный номер, полная ёмкость батареи.

## Circuit params

<img src="../1/ina_select.png" width="400">

- **Тип датчика:** INA219, INA226, INA231. INA231 может определяться как INA226 — нужно указать тип вручную
- **Коэффициенты коррекции** тока и напряжения (по эталонным приборам)
- **Макс. ток** (А)
- **Сопротивление шунта** (Ом). Для токов до 10 А — 0.005 Ом (5 мОм)
- **Интервал опроса** (по умолчанию 500 мс)
- **Power Strategy:** Battery / PowerSource (PowerSource — заглушка)

После изменения параметров схемы — перезагрузка.
