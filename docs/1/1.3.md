<a name="русский"></a>

## 🇷🇺 Русский

Количество вкладок соответствует числу обнаруженных датчиков при старте. Общие вкладки (Wi-Fi, Board, Updates) видны всегда, даже без датчиков.

### Wi-Fi settings

<img src="../1/wifi-change.png" width="400">

Смена SSID/пароля. ESP8266 — только 2.4 ГГц. При сохранении — перезагрузка.

### Board config

Частота шины I2C: от 100 кГц (по умолчанию) до 400 кГц. При изменении — перезагрузка.

### Updates

1. Указать URL к `ota.json`:
```json
{
  "version": "1.0.0",
  "buildNumber": 2,
  "esp8266": "http://example.com/ota/esp_8266_firmware.bin"
}
```
Ключ платформы: `esp8266` или `esp32` (строчные, без пробелов). URL бинарника — без редиректов.

2. Выбрать интервал проверки
3. Включить автообновление

МК сравнивает `buildNumber`. Если выше — скачивает и прошивает. При активном измерении на любом датчике OTA не запустится.

### Device info

Настройки наблюдаемого прибора для каждого датчика: название, ОС, серийный номер, полная ёмкость батареи.

### Circuit params

<img src="../1/ina_select.png" width="400">

- **Тип датчика:** INA219, INA226, INA231. INA231 может определяться как INA226 — нужно указать тип вручную
- **Коэффициенты коррекции** тока и напряжения (по эталонным приборам)
- **Макс. ток** (А)
- **Сопротивление шунта** (Ом). Для токов до 10 А — 0.005 Ом (5 мОм)
- **Интервал опроса** (по умолчанию 500 мс)
- **Power Strategy:** Battery / PowerSource (PowerSource — заглушка)

После изменения параметров схемы — перезагрузка.

### Гамбургер-меню

<img src="../1/hamburger-menu.png" width="400">

- Тема: светлая/тёмная
- Файловый менеджер (просмотр, загрузка, скачивание)
- Ручная загрузка бинарника для обновления

## 🇬🇧 English

The number of tabs corresponds to the number of sensors detected at startup. Common tabs (Wi-Fi, Board, Updates) are always visible, even without sensors.

### Wi-Fi settings

<img src="../1/wifi-change.png" width="400">

Change SSID/password. ESP8266 — 2.4 GHz only. Reboots after saving.

### Board config

I2C bus frequency: from 100 kHz (default) to 400 kHz. Reboots after changing.

### Updates

1. Specify the URL to `ota.json`:
```json
{
  "version": "1.0.0",
  "buildNumber": 2,
  "esp8266": "http://example.com/ota/esp_8266_firmware.bin"
}
```
Platform key: `esp8266` or `esp32` (lowercase, no spaces). Binary URL — no redirects.

2. Select the check interval
3. Enable auto-update

The MCU compares `buildNumber`. If it is higher, it downloads and flashes. OTA will not start if any sensor has an active measurement.

### Device info

Settings of the observed device for each sensor: name, OS, serial number, full battery capacity.

### Circuit params

<img src="../1/ina_select.png" width="400">

- **Sensor type:** INA219, INA226, INA231. INA231 may be detected as INA226 — the type must be set manually
- **Correction coefficients** for current and voltage (against reference instruments)
- **Max current** (A)
- **Shunt resistance** (Ω). For currents up to 10 A — 0.005 Ω (5 mΩ)
- **Polling interval** (default 500 ms)
- **Power Strategy:** Battery / PowerSource (PowerSource — stub)

Reboots after changing circuit parameters.

### Hamburger Menu

<img src="../1/hamburger-menu.png" width="400">

- Theme: light/dark
- File manager (view, upload, download)
- Manual binary upload for updates
